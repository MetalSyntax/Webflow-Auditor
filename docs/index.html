📦 Instalación

Recomendado · Todos los SO

Cargar descomprimida (Load Unpacked)

El método más simple y confiable. Funciona igual en macOS, Windows y Linux.

  1. Clona el repositorio:
    git clone https://github.com/TU-USUARIO/webflow-auditor.git
  2. Abre Chrome → chrome://extensions/
  3. Activa Modo desarrollador (toggle arriba a la derecha)
  4. Clic en "Cargar descomprimida" y selecciona la carpeta clonada
  5. Fija el ícono con el botón 🧩 de la barra
Ver repositorio →

Para actualizar: git pull en la carpeta + reload en chrome://extensions/

CRX · Linux / Enterprise

Archivo CRX (Self-hosted)

Instalación directa sin clonar el repo. Requiere modo desarrollador activo. En macOS Chrome 73+ puede estar bloqueado por política.

  1. Abre Chrome → chrome://extensions/
  2. Activa Modo desarrollador
  3. Descarga el archivo .crx y arrástralo sobre chrome://extensions/
  4. Confirma la instalación
⬇ Descargar .crx (v1.0.0) ⬇ Descargar .zip
⚠ macOS + Chrome 73+ Chrome bloquea instalación de CRX externos en macOS. Usa el método "Load Unpacked" en su lugar.

🔍 ¿Qué audita?

🔎 SEO (10 checks)

  • Title tag (longitud y presencia)
  • Meta description (50–160 chars)
  • H1 único y presente
  • Jerarquía de headings
  • Alt attributes en imágenes
  • SSL / HTTPS
  • Underscores en URLs
  • HTML lang attribute
  • iFrame titles
  • Meta keywords

⚡ Performance (8 checks)

  • DOM size (<1,500 elementos)
  • Meta viewport
  • Scripts render-blocking
  • document.write() detectado
  • Cache policy info
  • CLS (Cumulative Layout Shift)
  • GIFs animados
  • Font display

📱 Mobile (5 checks)

  • Meta viewport
  • Flash (obsoleto)
  • Tap targets ≥ 44×44px
  • Font size ≥ 12px
  • Scroll horizontal

♿ Accesibilidad (16 checks)

  • Button & link names
  • Image alt text
  • HTML lang válido (BCP47)
  • Duplicate IDs
  • Form labels
  • Tabindex values
  • aria-hidden con focusables
  • Skip link / Landmark regions
  • Meta refresh
  • Video captions
  • ARIA roles válidos
  • ARIA required children
  • List integrity
  • AccessKeys únicos

🔄 Cómo actualizar la extensión (equipo)

1
El desarrollador hace cambios y ejecuta el build script 
./build.sh
Elige el tipo de versión (patch/minor/major), el script actualiza manifest.json, genera el .crx y el .zip en docs/downloads/.
2
Sube el release a GitHub 
git add -A
git commit -m "release: v1.0.1"
git tag v1.0.1
git push && git push --tags
GitHub Pages publica automáticamente el nuevo .crx y esta página.
3
El equipo actualiza (método Load Unpacked) 
git pull   # dentro de la carpeta del repo
Luego en chrome://extensions/ → botón ↺ Reload en la tarjeta de la extensión.
